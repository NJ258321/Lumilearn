# 前端假数据与API断联分析报告

## 一、项目概述

本项目是一个智能学习辅助应用（LumiLearn），前端基于 React + TypeScript + Vite 构建，后端 API 已经封装完毕。前端页面中存在大量使用假数据（Mock Data）和定时器模拟的假交互，与真实 API 处于断联状态。

---

## 二、假数据来源

所有假数据统一存储在 `constants.ts` 文件中：

| 变量名 | 用途 | 行号 |
|--------|------|------|
| `MOCK_TASK_GROUPS` | 学习任务组（3个课程的学习任务） | 15-48 |
| `MOCK_COURSES` | 课程列表（8门课程） | 50-59 |
| `MOCK_TRANSCRIPT` | 课堂文字稿（10个片段） | 61-72 |

## 五、修复优先级建议

### P0 - 高优先级（影响核心功能）

1. **PracticeList.tsx**
   - 将 `courseId: 'c1'` 改为动态获取
   - 将 `displayCourses` 硬编码数据改为调用 `getCourseList()` + `getKnowledgeMastery()`

2. **Dashboard.tsx**
   - 将 `MOCK_TASK_GROUPS` 改为调用 `getDailyRecommendation()` 或 `getDashboard()`
   - 将 `setTimeout` 模拟改为调用 `optimizeReview()` API

3. **TimeMachine.tsx**
   - 将 `MOCK_TRANSCRIPT` 改为调用 `getStudyRecordById()`
   - 将硬编码的 `studyRecordId` 改为必填参数

### P1 - 中优先级（数据一致性）

1. **Recorder.tsx**
   - 将 `MOCK_COURSES.filter` 改为使用已获取的 `courses` 状态

2. **Analysis.tsx**
   - 将 `MOCK_COURSES` 改为调用 `getCourseList()`
   - 将 `weakPoints` 改为调用 `getBottlenecks()`

3. **ExamCalendar.tsx**
   - 将 `mockPlans` 改为调用 `generateReviewPlan()`

### P2 - 低优先级（UI展示优化）

1. **CourseDetailReview.tsx**
   - 将 `INITIAL_DATA` 改为调用 `getChapterList()` + `getKnowledgePointList()`

2. **各组件中的 setTimeout**
   - 逐步替换为真实的异步操作
