### P3.1: 扩展 Schema 知识图谱关系

**任务ID**: P3.1
**文件**: `code/backend/prisma/schema.prisma`
**输出**: 新增 KnowledgeRelation 模型

**当前状态**: ✅ 已完成

**功能说明**:
- 新增知识点关系模型，支持前后置知识点关联
- 支持多种关系类型：PREREQUISITE（前置）、RELATED（相关）、EXTENDS（扩展）、EXAMPLE（示例）

**新增模型**:
```prisma
model KnowledgeRelation {
  id             String   @id @default(uuid())
  sourceId       String   // 源知识点ID
  targetId       String   // 目标知识点ID
  relationType   String   // PREREQUISITE | RELATED | EXTENDS | EXAMPLE
  weight         Int      @default(5) // 关系权重 1-10
  createdAt      DateTime @default(now())

  source         KnowledgePoint @relation("KnowledgeRelationSource", fields: [sourceId], references: [id], onDelete: Cascade)
  target         KnowledgePoint @relation("KnowledgeRelationTarget", fields: [targetId], references: [id], onDelete: Cascade)

  @@unique([sourceId, targetId, relationType])
  @@index([sourceId])
  @@index([targetId])
  @@index([relationType])
}
```

**relationType 说明**:
| 类型 | 说明 |
|------|------|
| PREREQUISITE | 前置知识点（必须先学）|
| RELATED | 相关知识点 |
| EXTENDS | 扩展知识点 |
| EXAMPLE | 示例关系 |

**预估时间**: 1小时

        "weight": 8

    "prerequisites": [

### P3.4: 知识点解释生成 API

**任务ID**: P3.4
**接口**: `POST /api/ai/explain`
**文件**: 新建 `code/backend/src/routes/ai.ts`

**当前状态**: ✅ 已完成

**功能说明**:
- 调用 Gemini API 生成知识点解释
- 结合知识点详情和关联知识生成个性化解释
- 返回 Markdown 格式解释内容
- 支持 brief/detailed/example 三种风格

**API 接口详情**:

#### 请求
`POST /api/ai/explain`

**请求体**:
```json
{
  "knowledgePointId": "uuid",
  "style": "detailed",  // brief | detailed | example
  "includeRelated": true,
  "includeExamples": true
}
```

**参数说明**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| knowledgePointId | string | 是 | 知识点ID |
| style | string | 否 | 解释风格: brief/detailed/example |
| includeRelated | boolean | 否 | 是否包含关联知识，默认 true |
| includeExamples | boolean | 否 | 是否包含示例，默认 true |

**响应格式**:
```json
{
  "success": true,
  "data": {
    "knowledgePoint": {
      "id": "uuid",
      "name": "微分中值定理"
    },
    "explanation": "## 微分中值定理\n\n### 定义\n\n拉格朗日中值定理是...",
    "relatedPoints": [
      { "id": "uuid", "name": "导数" },
      { "id": "uuid", "name": "极限" }
    ],
    "examples": [
      { "question": "例题1", "answer": "解答1" }
    ],
    "generatedAt": "2026-03-05T10:00:00Z"
  }
}
```

**Prompt 模板**:
```
你是一位大学数学教授，请用{{style}}风格解释以下知识点：

知识点：{{knowledgePointName}}
章节：{{chapterName}}
课程：{{courseName}}
掌握程度：{{masteryScore}}%

{{#if includeRelated}}
关联知识点：
{{#each relatedPoints}}
- {{name}} (掌握度：{{masteryScore}}%)
{{/each}}
{{/if}}

请生成：
1. 核心概念解释
2. 关键公式
3. 常见误区
{{#if includeExamples}}
4. 典型例题
{{/if}}
```

**实现要点**:
1. 查询知识点详情及关联信息
2. 构建 Prompt 模板
3. 调用 Gemini 服务
4. 缓存解释结果（可选）

### P3.6: 薄弱点智能发现算法

**任务ID**: P3.6
**接口**: `POST /api/ai/analyze-weak-points`
**文件**: `code/backend/src/routes/ai.ts`

**当前状态**: ✅ 已完成

**功能说明**:
- 基于错题率、复习时间、知识掌握度等多维度分析
- 自动识别薄弱知识点
- 生成薄弱点分析报告
- 支持 quick/comprehensive 两种分析深度

**API 接口详情**:

#### 请求
`POST /api/ai/analyze-weak-points`

**请求体**:
```json
{
  "courseId": "uuid",
  "analyzeDepth": "comprehensive"  // quick | comprehensive
}
```

**参数说明**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| courseId | string | 是 | 课程ID |
| analyzeDepth | string | 否 | 分析深度: quick/comprehensive |

**响应格式**:
```json
{
  "success": true,
  "data": {
    "courseId": "uuid",
    "totalKnowledgePoints": 50,
    "weakPoints": [
      {
        "knowledgePoint": {
          "id": "uuid",
          "name": "泰勒公式",
          "masteryScore": 35,
          "status": "WEAK"
        },
        "analysis": {
          "mistakeRate": 0.6,
          "reviewMissCount": 3,
          "lastReviewDate": "2026-02-20",
          "avgTimeSpent": 15,
          "errorPatterns": [
            "展开项数不足",
            "收敛区间判断错误"
          ]
        },
        "priority": "high",
        "suggestedAction": "重点练习泰勒公式展开"
      }
    ],
    "learningInsights": [
      "你的薄弱点主要集中在需要综合应用的知识点",
      "建议加强公式推导过程的练习"
    ],
    "generatedAt": "2026-03-05T10:00:00Z"
  }
}
```

**算法逻辑**:
```
薄弱点评分 = 错题权重 * 错题率 + 时间权重 * (未复习天数) + 掌握度权重 * (100 - 掌握度)

权重配置:
- 错题率权重: 0.4
- 未复习天数权重: 0.3
- 掌握度权重: 0.3
```

**判定规则**:
| 状态 | 条件 |
|------|------|
| HIGH (高优先级) | 评分 >= 70 或 错题率 > 50% |
| MEDIUM (中优先级) | 评分 >= 40 或 30天未复习 |
| LOW (低优先级) | 其他 |

**预估时间**: 4小时
