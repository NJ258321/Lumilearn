### P1.1: 实现文件删除 API

**任务ID**: P1.1
**接口**: `DELETE /api/upload/:filename`
**文件**: `code/backend/src/routes/upload.ts`

**当前状态**: ✅ 已完成

**功能说明**:
- 根据文件名删除已上传的文件
- 验证文件存在性
- 返回删除结果

**实现要点**:
1. 使用 `fs.unlink` 删除物理文件
2. 验证文件路径安全性（防止路径遍历攻击）

**预估时间**: 30分钟

### P1.2: 实现文件预览/获取 API

**任务ID**: P1.2
**接口**: `GET /api/upload/:filename`
**文件**: `code/backend/src/routes/upload.ts`

**当前状态**: ✅ 已完成

**新增功能**:
- 支持图片直接显示（inline）
- 支持音频/视频流式播放（Range 请求）
- 支持文档下载
- 根据文件扩展名自动识别 Content-Type
- 缓存头设置

**功能说明**:
- 根据文件名获取并预览文件
- 支持图片直接显示、文档下载
- 设置正确的 Content-Type

**实现要点**:
1. 根据文件扩展名设置 Content-Type
2. 支持 Range 请求（视频/音频流式播放）
3. 文件不存在时返回 404

**支持的文件类型**:
| 类型 | Content-Type |
|------|--------------|
| 图片 | image/jpeg, image/png, image/gif, image/webp |
| 音频 | audio/mpeg, audio/wav, audio/ogg |
| 视频 | video/mp4, video/webm |
| 文档 | application/pdf |

**预估时间**: 30分钟

### P1.4: 统一错误处理

**任务ID**: P1.4
**接口**: 全局中间件
**文件**: `code/backend/src/routes/upload.ts`

**当前状态**: ✅ 已完成

**已完成功能**:
- 统一的错误响应格式 `{ success: false, error: string, code: string }`
- 新增错误码：
  - `INVALID_FILENAME` - 无效的文件名
  - `FILE_NOT_FOUND` - 文件不存在
  - `FILE_TOO_LARGE` - 文件过大
  - `INVALID_FILE_TYPE` - 文件类型不支持
  - `FILE_DELETE_FAILED` - 文件删除失败
  - `FILE_READ_FAILED` - 文件读取失败

**功能说明**:
- 统一 API 错误响应格式
- 文件操作专用错误类型
- 全局错误日志

**错误响应格式**:
```typescript
// 成功
{ success: true, data: ... }

// 客户端错误 (4xx)
{ success: false, error: '错误描述', code: 'ERROR_CODE' }

// 服务端错误 (5xx)
{ success: false, error: '服务器错误', code: 'INTERNAL_ERROR' }
```

**新增错误码**:
| 错误码 | 说明 | HTTP状态 |
|--------|------|----------|
| FILE_NOT_FOUND | 文件不存在 | 404 |
| FILE_TOO_LARGE | 文件过大 | 413 |
| INVALID_FILE_TYPE | 文件类型不支持 | 400 |
| FILE_DELETE_FAILED | 文件删除失败 | 500 |
| FILE_READ_FAILED | 文件读取失败 | 500 |

**预估时间**: 30分钟
