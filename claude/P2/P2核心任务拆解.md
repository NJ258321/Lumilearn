## 二、任务拆解详情

### P2.1: 音频元数据提取 API

**任务ID**: P2.1
**接口**: `GET /api/audio/:id/metadata`
**文件**: 新建 `code/backend/src/routes/audio.ts`

**当前状态**: ✅ 已完成

**功能说明**:
- 根据学习记录 ID 获取音频元数据
- 返回时长、格式、采样率、比特率等信息
- 关联的知识点和时间标记

**请求参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 学习记录 ID |

**响应格式**:
```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "title": "学习记录标题",
    "audioUrl": "/uploads/audio-xxx.mp3",
    "duration": 3600,
    "format": "mp3",
    "bitrate": 128000,
    "sampleRate": 44100,
    "channels": 2,
    "size": 52428800,
    "createdAt": "2026-03-05T10:00:00Z",
    "knowledgePoints": [
      { "id": "uuid", "name": "知识点名称" }
    ],
    "timeMarksCount": 10
  }
}
```

**实现要点**:
1. 查询 StudyRecord 获取音频 URL
2. 使用 `music-metadata` 或 `ffprobe` 获取音频元数据
3. 关联查询知识点和时间标记数量
4. 缓存元数据结果

**预估时间**: 2小时

### P2.2: 音频转写接口（预留）

**任务ID**: P2.2
**接口**: `POST /api/audio/:id/transcribe`
**文件**: `code/backend/src/routes/audio.ts`

**当前状态**: ✅ 已完成（预留框架）

**已完成功能**:
- 预留框架就绪
- 支持配置多种语音识别服务（阿里云、讯飞、Google）
- 额外接口：`GET /api/audio/transcribe/status` 获取服务状态

**功能说明**:
- 调用外部语音识别服务进行音频转写
- 返回文本内容和时间戳对应关系

**请求参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 学习记录 ID |
| language | string | 否 | 语言代码，默认 "zh-CN" |

**请求体**:
```json
{
  "language": "zh-CN"
}
```

**响应格式**:
```json
{
  "success": true,
  "data": {
    "text": "转写的文本内容...",
    "segments": [
      {
        "start": 0.0,
        "end": 5.5,
        "text": "这是第一句话"
      },
      {
        "start": 5.5,
        "end": 10.2,
        "text": "这是第二句话"
      }
    ],
    "language": "zh-CN"
  }
}
```

**实现要点**:
1. 预留接口框架，暂时返回提示信息
2. 后续可集成阿里云语音识别、讯飞语音识别等
3. 支持异步处理（返回任务 ID）

**预估时间**: 1小时（框架）/ 预留

### P2.3: 音频片段提取 API

**任务ID**: P2.3
**接口**: `GET /api/audio/:id/segment`
**文件**: `code/backend/src/routes/audio.ts`

**当前状态**: ✅ 已完成

**已完成功能**:
- 根据时间范围提取音频片段
- 使用 ffmpeg 进行音频处理
- 最大片段时长限制（5分钟）
- 流式返回音频文件
- 自动清理临时文件

**功能说明**:
- 根据时间范围提取音频片段
- 支持下载或流式传输

**请求参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 学习记录 ID |
| start | number | 是 | 开始时间（秒）|
| end | number | 是 | 结束时间（秒）|

**响应**:
- 返回音频片段文件流
- Content-Type: audio/mpeg
- Content-Disposition: attachment; filename="segment.mp3"

**错误响应**:
```json
{
  "success": false,
  "error": "时间范围无效",
  "code": "INVALID_TIME_RANGE"
}
```

**实现要点**:
1. 使用 `ffmpeg` 提取音频片段
2. 验证时间范围有效性
3. 流式返回处理后的音频
4. 限制最大片段时长（如 5 分钟）

**预估时间**: 2小时

### P2.4: 语音活动检测接口（可选）

**任务ID**: P2.4
**接口**: `POST /api/audio/:id/vad`
**文件**: `code/backend/src/routes/audio.ts`

**当前状态**: ✅ 已完成

**已完成功能**:
- 基于 ffmpeg 音量分析检测语音段
- 支持灵敏度、最小语音时长、最小静音间隔参数
- ffmpeg 不可用时返回统计估算结果
- 额外接口：`GET /api/audio/vad/status` 获取服务状态

**功能说明**:
- 检测音频中的语音活动
- 返回语音片段的时间范围
- 用于自动标记重点区域

**请求参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 学习记录 ID |
| sensitivity | number | 否 | 灵敏度 0-100，默认 50 |

**响应格式**:
```json
{
  "success": true,
  "data": {
    "segments": [
      { "start": 0.5, "end": 10.2 },
      { "start": 15.0, "end": 25.5 },
      { "start": 30.0, "end": 45.0 }
    ],
    "totalSpeechDuration": 35.2,
    "audioDuration": 3600
  }
}
```

**实现要点**:
1. 使用 Web Audio API 或服务端 VAD 库
2. 预留框架，可后续集成
3. 返回语音片段用于自动生成时间标记

**预估时间**: 2小时（预留框架）
